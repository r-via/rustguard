## Codebase Patterns

- **no_std stubs in rustguard-core**: The crate has `#[cfg(not(feature = "std"))]` stubs for functions like `fill_random`, `elapsed_since`, and `now`. The kernel module (`rustguard-kmod`) is outside the Cargo workspace and builds separately via Kbuild/Makefile — it has its own implementations. No Rust toolchain in this environment; cannot run `cargo build`/`cargo test`.
- **Feature gating**: `rustguard-core` uses `std` as default feature. All workspace members use default features, so no_std paths are never active in normal workspace builds.
- **Signal handling pattern**: `ctrlc_handler(f: impl FnOnce())` + `signal_noop` exists in `rustguard-daemon/src/tunnel.rs` (private). Uses sigwait in a dedicated thread. Block SIGINT/SIGTERM before spawning workers. `libc` is available in both `rustguard-daemon` and `rustguard-enroll`.
- **Mutex before I/O**: Scope Mutex guards in block expressions to release before syscalls (send_to, write). Pattern: `let val = { let g = mutex.lock(); compute(g) }; io(val);`
- **`_at` pattern for no_std**: Time-dependent methods have `#[cfg(not(feature = "std"))]` `_at(now_ns: u64)` variants where the caller provides a nanosecond monotonic timestamp. Use `elapsed_from_ns(stored_ns, now_ns) -> Duration` helper with `saturating_sub`.
- **Raw FD cleanup pattern**: When acquiring multiple raw i32 file descriptors sequentially, use `.map_err(|e| { unsafe { libc::close(fd); } e })?` on each subsequent fallible step to close earlier fds on failure. Raw i32 has no Drop — XdpProgram's Drop only runs on successful construction.
- **`#[repr(C)]` padding**: Always verify `size_of::<Struct>()` matches your `_pad` calculation. Structs with u64 fields have alignment 8; compiler silently adds trailing padding bytes that may be uninitialized when passed to kernel syscalls.
- **UnsafeCell for io_uring buffers**: When a struct needs `&self -> *mut u8` (e.g., for io_uring SQEs), wrap the data in `UnsafeCell<Vec<u8>>`. Use `get_mut()` (safe) from `&mut self` methods, `unsafe { &*self.data.get() }` from `&self` read methods, and `unsafe { (*self.data.get()).as_mut_ptr() }` for raw pointer derivation.
