### rustguard-crypto

The cryptographic primitive layer. Contains no protocol logic. All functions operate on byte slices; no heap allocation in no_std mode. Key types: X25519SecretKey, X25519PublicKey, ChaCha20Poly1305, Blake2sHmac, Hkdf, Tai64n.

### rustguard-core

Protocol state machine implementing the WireGuard Noise_IKpsk2 handshake and transport session lifecycle. Handles the handshake message wire format (initiation, response, cookie reply), the 2048-bit sliding-window anti-replay bitmap, and the timer state machine (rekey, keepalive, session expiry). Produces and consumes opaque byte buffers; has no I/O dependency.

### rustguard-tun

Platform I/O layer. Abstracts TUN device creation and packet I/O behind a common trait. Implementations:

- **UtunDevice** — macOS utun via kernel control socket (AF_SYSTEM)
- **LinuxTunDevice** — Linux /dev/net/tun with IFF_TUN | IFF_NO_PI
- **MultiQueueTun** — Linux multi-queue TUN for parallel I/O
- **AfXdpSocket** — AF_XDP zero-copy path via UMEM ring buffers
- **IoUringTun** — io_uring-backed async TUN I/O

### rustguard-daemon

Reads a wg.conf-compatible config file, configures peers and allowed IPs, then runs the tunnel loop. Handles IPv4/IPv6 dual-stack routing and OS route table management on both Linux and macOS. Provides clean shutdown with signal handling and route cleanup.

### rustguard-enroll

Implements the enrollment protocol: a token-derived XChaCha20 key encrypts the initial key exchange, a CIDR-based IP pool allocator assigns peer addresses (server always gets .1), and a pairing window enforces physical-presence enrollment. State is serialized to JSON for restart persistence.

### rustguard-kmod
