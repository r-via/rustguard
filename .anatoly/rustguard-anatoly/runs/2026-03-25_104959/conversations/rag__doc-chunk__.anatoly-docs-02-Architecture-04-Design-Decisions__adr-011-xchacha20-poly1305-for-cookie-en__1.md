## User

Section "ADR-011 — XChaCha20-Poly1305 for Cookie Encryption" from `.anatoly/docs/02-Architecture/04-Design-Decisions.md`:

**Status:** Accepted

**Context:** WireGuard's cookie reply message must be encrypted with a key derived from a server-side rotating secret and the initiator's source IP/port. A 24-byte nonce is desirable to allow random nonce generation without collision risk. ChaCha20-Poly1305 uses a 12-byte nonce, which is too short for random generation.

**Decision:** Cookie reply encryption uses XChaCha20-Poly1305, which extends the nonce to 24 bytes using the HChaCha20 sub-key derivation. Nonces for cookie replies are generated from the CSPRNG.

**Consequences:**
- Random nonce generation is safe for cookie replies (collision probability negligible under 2^80 operations).
- The encryption used for cookie replies is distinct from transport encryption; the distinction is documented in code comments.
