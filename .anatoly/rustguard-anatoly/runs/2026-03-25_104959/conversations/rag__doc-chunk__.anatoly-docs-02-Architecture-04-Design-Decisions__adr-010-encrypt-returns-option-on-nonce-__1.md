## User

Section "ADR-010 — `encrypt()` Returns `Option` on Nonce Exhaustion" from `.anatoly/docs/02-Architecture/04-Design-Decisions.md`:

**Status:** Accepted (security improvement)

**Context:** ChaCha20-Poly1305 supports a 64-bit nonce counter. Exhausting the nonce space (2^64 messages) with the same session key would enable an attacker to observe key-stream reuse. The original implementation panicked on nonce overflow, which would crash the daemon in production.

**Decision:** encrypt() returns Option and returns None when the nonce counter would overflow. The transport session layer treats None as a signal to force a rekey before any further encryption.

**Consequences:**
- No nonce reuse is possible at the encryption API boundary.
- The daemon does not crash; it initiates a new handshake instead.
- The WireGuard timer-driven rekey (after 2^60 messages) remains the primary guard; Option is a last-resort safety net.
