Section "rustguard-core" from `.anatoly/docs/04-API-Reference/03-Types-and-Interfaces.md`:

**Crate:** rustguard-core · **no_std compatible**

### HandshakeState

Ephemeral state held during an in-progress Noise_IK handshake. All sensitive fields are zeroed on drop.

HandshakeState implements ZeroizeOnDrop. It must not be retained after HandshakeState::into_session() consumes it to produce a TransportSession.

### TransportSession

An established, keyed WireGuard session between two peers.

### ReplayWindow

A 2048-bit sliding bitmap that provides anti-replay protection for transport packets.

**Important:** The check and update steps are intentionally separated. check() must succeed before AEAD verification; update() must only be called after the AEAD tag passes. Calling update() before verification would allow garbage counters to advance the window.

### CookieChecker

Server-side DoS rate-limiting state. Maintains a rotating 32-byte secret used to generate MAC2 cookies. The secret rotates on a fixed schedule so stale cookies are automatically invalidated.

CookieChecker validates mac1 before any Diffie-Hellman computation, ensuring unauthenticated packets cannot burn CPU.

### CookieState

Client-side cookie storage. Holds the most recently received server cookie and its expiry.

When the server is under load and requires MAC2, the client uses the stored cookie to construct mac2 in the next handshake initiation.

### WireMessage

Discriminated enum over the four WireGuard wire message types.

### TimerState

State machine tracking per-session timer events.

Timers are evaluated against TransportSession::created_at and send_counter. Retry attempts include jitter to avoid thundering-herd reconnects.

### AllowedIps

An IP routing table entry allowing both IPv4 and IPv6 CIDR prefixes.

Each peer holds a Vec used for packet routing decisions in both TUN-read and UDP-receive paths.
