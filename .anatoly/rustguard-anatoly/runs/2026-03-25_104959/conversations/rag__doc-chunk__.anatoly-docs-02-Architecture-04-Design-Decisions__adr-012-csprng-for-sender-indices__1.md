## User

Section "ADR-012 — CSPRNG for Sender Indices" from `.anatoly/docs/02-Architecture/04-Design-Decisions.md`:

**Status:** Accepted (correctness fix)

**Context:** The initial implementation used the nanosecond timestamp as the 32-bit sender index in handshake messages. Timestamps are not unique under concurrent handshakes and are predictable, enabling index guessing attacks.

**Decision:** Sender indices are drawn from the operating system CSPRNG (via the getrandom crate). Collision probability for 32-bit random values across active sessions is negligible for all realistic peer counts.

**Consequences:**
- Sender indices are unpredictable to off-path attackers.
- No coordination or global counter is required.
