## User

Section "ADR-001 — Clean-Room Implementation (No libwg)" from `.anatoly/docs/02-Architecture/04-Design-Decisions.md`:

**Status:** Accepted

**Context:** Existing WireGuard userspace tools (wireguard-go, boringtun) wrap libwg or the official Go implementation. Using a binding would deliver a working tunnel faster but would make it impossible to deeply understand the protocol internals or push the implementation in directions the upstream library does not support.

**Decision:** Implement every layer from scratch — cryptographic primitives, Noise handshake, wire format, TUN device management, timer state machine — with no dependency on any existing WireGuard library.

**Consequences:**
- All protocol logic is auditable and modifiable within the repository.
- Interoperability must be validated manually against the WireGuard specification rather than inherited from a reference library.
- Several bugs were introduced that a reference library would have prevented (see ADR-005 through ADR-010); all were found and fixed during the Phase 2 security audit.
