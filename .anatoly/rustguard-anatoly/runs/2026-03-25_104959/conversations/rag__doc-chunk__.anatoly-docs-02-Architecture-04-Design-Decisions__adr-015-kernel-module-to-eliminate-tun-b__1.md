## User

Section "ADR-015 — Kernel Module to Eliminate TUN Bottleneck" from `.anatoly/docs/02-Architecture/04-Design-Decisions.md`:

**Status:** Accepted

**Context:** Userspace WireGuard requires every packet to cross the kernel–userspace boundary twice (TUN read + UDP write for egress; UDP read + TUN write for ingress). At high throughput this boundary becomes the dominant cost. AF_XDP reduces copies but does not eliminate the boundary crossings entirely.

**Decision:** rustguard-kmod implements the complete data path inside the Linux kernel as an out-of-tree module targeting kernel 6.10+. The cryptographic and protocol logic is compiled from rustguard-crypto and rustguard-core (both no_std) directly into the module. No TUN device is involved.

**Consequences:**
- The kernel–userspace boundary is eliminated for the data path; packets are encrypted/decrypted in-kernel.
- The module is out-of-tree and requires kernel headers for the target kernel version to build.
- The minimum supported kernel is 6.10 (required for stable Rust-in-kernel ABI support).
