## User

Section "Security-Critical Patterns" from `.anatoly/docs/05-Development/03-Code-Conventions.md`:

These patterns were introduced or hardened during the Phase 2 audit. Deviating from them is a security defect, not a style issue.

### 1. Authenticate Before Compute

Expensive operations (DH, key derivation) must never execute on unauthenticated input. Always verify MAC1 first.

### 2. Constant-Time Comparisons

All equality checks on secret or authentication data must use subtle::ConstantTimeEq. Never use ==, !=, or any hand-rolled loop.

### 3. Zeroize Sensitive State on Drop

Any struct holding keying material (chaining keys, handshake hashes, PSK) must derive ZeroizeOnDrop. This applies to all types in rustguard-crypto and rustguard-core that own secret bytes.

### 4. Replay Window: Check Before Update

The anti-replay window must be checked against the incoming counter *before* authenticating the packet and *updated* only after successful AEAD verification. Never update the window on unauthenticated input.

### 5. Nonce Exhaustion — Return, Don't Panic

encrypt() must return Option> (or an appropriate Result) when the nonce counter is exhausted. Panicking in a network path is a denial-of-service vector.
