Section "IPv6 Dual-Stack" from `.anatoly/docs/03-Guides/02-Advanced-Configuration.md`:

AllowedIPs accepts both IPv4 and IPv6 CIDR ranges in the same peer stanza. The tunnel inspects the IP version byte of each plaintext packet to route it correctly.

TUN configuration on both platforms supports dual-stack:

- **macOS** — ifconfig inet6 is used to add the IPv6 address to the utun interface.
- **Linux** — ip -6 addr add is used. Routes for both families are inserted and cleaned up on shutdown.

The server's --pool flag currently accepts a single CIDR per invocation. To use both families, run two servers on different ports or supply AllowedIPs entries manually via wg.conf mode.
