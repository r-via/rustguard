Section "Handshake Protocol" from `.anatoly/docs/02-Architecture/02-Core-Concepts.md`:

### Noise\_IKpsk2

The specific Noise Protocol Framework pattern used by WireGuard. I = the initiator's static key is transmitted (encrypted) during initiation. K = the responder's static key is pre-known to the initiator. psk2 = an optional pre-shared symmetric key is mixed in at pattern message 2. The handshake produces two session keys: one for each direction of transport.

Three message types exist on the wire:

### Chaining Key and Handshake Hash

Two mutable state values evolved throughout the Noise handshake. The chaining key accumulates entropy from each DH operation; the handshake hash commits to the transcript. Both are zeroed on drop via ZeroizeOnDrop to limit key material lifetime in memory.

### Sender Index

A 32-bit identifier chosen by the initiating peer to tag its handshake and associate subsequent transport packets with a session. Originally generated with a nanosecond clock hack; replaced with a CSPRNG (getrandom crate) to prevent predictability.
