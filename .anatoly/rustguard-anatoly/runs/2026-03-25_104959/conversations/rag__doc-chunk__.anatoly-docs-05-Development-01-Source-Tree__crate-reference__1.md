### rustguard-crypto

**Design notes:**
- no_std with an optional std feature flag. All code compiles for bare-metal targets and is reused verbatim in rustguard-kmod.
- encrypt() returns Option rather than panicking on nonce exhaustion (introduced in the security audit phase).
- All secret key types implement ZeroizeOnDrop via the zeroize crate.
- Constant-time comparisons use subtle::ConstantTimeEq — no hand-rolled black_box usage.

### rustguard-core

**Design notes:**
- no_std with optional std feature, mirroring rustguard-crypto.
- The replay window is split into check() (read-only, pre-decrypt) and update() (mutating, post-decrypt) to prevent window poisoning from garbage counters.
- CookieChecker rotates server secrets and issues Cookie Reply (message type 3). CookieState tracks the last received cookie for MAC2 computation.
- MAC1 verification is performed before any DH operation to prevent unauthenticated CPU burn.
- Handshake chaining key, hash, and PSK state implement ZeroizeOnDrop.

### rustguard-tun

**Design notes:**
- All file descriptors are opened with O_CLOEXEC.
- ifreq padding on Linux is 20 bytes (not 12) to correctly read kernel-returned MTU.
- AF_XDP and io_uring paths are behind Cargo feature flags; the default build uses the standard TUN path.
- The BPF loader in bpf.rs attaches programs that steer RX traffic to AF_XDP UMEM queues.

### rustguard-daemon

**Design notes:**
- AllowedIPs parser handles both IPv4 and IPv6 CIDR ranges.
- Route installation and cleanup are symmetric: the signal handler reverses all routes added at startup.
- rustguard up wg0.conf remains fully functional alongside the zero-config enrollment path.

### rustguard-enroll

**Design notes:**
- Token-derived key: the shared token string is used to derive an XChaCha20-Poly1305 encryption key for the enrollment exchange; no pre-shared asymmetric keys are required.
- The pairing window is managed by an atomic timestamp. rustguard open  sets the deadline; rustguard close zeroes it immediately.
- Enrollment is closed by default. Existing peers are unaffected when the window closes.
- State survives daemon restarts — assigned IPs and peer public keys are reloaded from state.json on startup.

### rustguard-kmod

**Design notes:**
- rustguard-crypto and rustguard-core are consumed directly as no_std dependencies — no duplication of cryptographic or protocol logic.
- The C shim wraps kernel-internal APIs that lack Rust bindings in the 6.10 kernel tree. The shim surface is deliberately minimal.
- The kernel module eliminates the TUN overhead present in the userspace daemon by operating as a native net_device driver.
