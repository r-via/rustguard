## User

Section "Cryptographic Primitives" from `.anatoly/docs/02-Architecture/02-Core-Concepts.md`:

RustGuard implements every cryptographic primitive in rustguard-crypto, which targets both std and no_std environments.

### X25519

Elliptic-curve Diffie-Hellman key exchange over Curve25519. Used during the Noise_IK handshake to derive a shared secret from a static keypair and an ephemeral keypair. Each peer has a long-lived static X25519 keypair and generates a fresh ephemeral keypair for every handshake initiation.

### ChaCha20-Poly1305

Authenticated encryption with associated data (AEAD) used for all transport-layer packet encryption. Provides both confidentiality and integrity. The nonce is derived from the session counter; nonce exhaustion (at 2⁶⁰ messages) triggers a rekey rather than wrapping — encrypt() returns Option to signal exhaustion without panicking.

### XChaCha20-Poly1305

An extended-nonce variant of ChaCha20-Poly1305. Used specifically for cookie encryption in the DoS-protection layer, where a 192-bit nonce eliminates the nonce-reuse risk present in the base construction.

### HMAC-BLAKE2s

RFC 2104 HMAC construction over BLAKE2s — the ipad/opad double-hash form. Used as the pseudorandom function (PRF) within WireGuard's HKDF-based key derivation. Note: keyed BLAKE2s alone is **not** equivalent and produces non-interoperable output.

### HKDF

HMAC-based key derivation function. Chains HMAC-BLAKE2s calls to expand a shared secret into the chaining key, handshake hash, and session keys used for transport encryption.

### TAI64N

International Atomic Time with nanosecond precision, encoded as a 12-byte big-endian value. Embedded in handshake initiation messages and validated on receipt to prevent timestamp replay attacks (stale initiations must be rejected).
