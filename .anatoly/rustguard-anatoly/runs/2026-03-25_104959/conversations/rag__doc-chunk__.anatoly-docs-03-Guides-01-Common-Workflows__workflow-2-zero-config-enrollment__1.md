Section "Workflow 2: Zero-Config Enrollment" from `.anatoly/docs/03-Guides/01-Common-Workflows.md`:

The enrollment workflow eliminates manual key exchange and IP assignment. It is suited for homelabs, dynamic device fleets, and any scenario where operational simplicity is preferred.

### 2.1 Start the Enrollment Server

On the machine that will act as the server (must be reachable at a known address):

- The server is assigned the first address in the pool (.1).
- Enrollment starts **closed**. No clients can join until the window is opened.
- The server listens for peer connections on the default WireGuard port (51820).
- A UNIX domain control socket is opened for runtime management.

### 2.2 Open the Enrollment Window

On the server, open the enrollment window for a time-limited period. This models physical-presence authorization (similar to Zigbee pairing).

The window closes automatically after the specified duration. To close it immediately:

### 2.3 Enroll a Client

On the client machine, within the enrollment window:

- The token is used to derive an XChaCha20 encryption key for the enrollment exchange.
- The server's IP pool allocator assigns the next available address to the client.
- No config file is written or required on either side.

Multiple clients can enroll sequentially or within the same window. Each receives a unique IP address from the pool.

### 2.4 Check Server Status

Outputs the current enrollment window state (open/closed, seconds remaining) and the number of enrolled peers.
