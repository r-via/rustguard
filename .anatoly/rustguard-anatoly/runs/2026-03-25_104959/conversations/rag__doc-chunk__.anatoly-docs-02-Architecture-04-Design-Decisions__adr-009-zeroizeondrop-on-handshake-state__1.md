## User

Section "ADR-009 — `ZeroizeOnDrop` on Handshake State" from `.anatoly/docs/02-Architecture/04-Design-Decisions.md`:

**Status:** Accepted (security improvement)

**Context:** Handshake state (chaining key, running hash, PSK) is held in heap or stack memory. Without explicit zeroing, sensitive key material persists in memory after the handshake completes or fails, where it could be recovered from process memory dumps or swap.

**Decision:** All handshake state structs derive or implement ZeroizeOnDrop. Key material is overwritten with zeros when the struct is dropped, whether the handshake succeeded or not.

**Consequences:**
- Forward secrecy properties hold at the memory level as well as the protocol level.
- Session keys and ephemeral private keys are not recoverable from a post-session memory snapshot.
