Section "Deployment Modes" from `.anatoly/docs/02-Architecture/01-System-Overview.md`:

### Userspace Mode

rustguard-daemon and rustguard-enroll both run entirely in userspace. Encrypted packets are exchanged via a UDP socket; plaintext packets are injected and read from a TUN device file descriptor. rustguard-tun abstracts the platform-specific TUN API and provides multiple I/O backends (blocking read/write, multi-queue, AF_XDP zero-copy, io_uring).

### Kernel Module Mode

rustguard-kmod eliminates the TUN device overhead by processing packets directly inside the Linux network stack. It reuses rustguard-crypto and rustguard-core unchanged (both compiled as no_std) and replaces the TUN file descriptor with direct sk_buff / xdp_buff access via the kernel's netdev layer.

### Enrollment Mode

rustguard-enroll adds a runtime control plane over rustguard-core. A UNIX domain socket accepts open, close, and status commands while the tunnel is live. Peer state (public keys, assigned IPs) is persisted to ~/.rustguard/state.json.
