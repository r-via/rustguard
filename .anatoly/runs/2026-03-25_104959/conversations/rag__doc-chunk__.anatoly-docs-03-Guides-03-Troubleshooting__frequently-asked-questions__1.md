Section "Frequently Asked Questions" from `.anatoly/docs/03-Guides/03-Troubleshooting.md`:

**Can rustguard up and rustguard serve run simultaneously?**

Yes. rustguard up runs the tunnel in standard wg.conf mode; rustguard serve runs the zero-config enrollment server. They operate independently. The UNIX domain control socket used by rustguard open, rustguard close, and rustguard status communicates only with the enrollment server process.

**Does the enrollment server require open enrollment to maintain existing peer connections?**

No. The enrollment window controls only whether *new* peers can join. Existing peers remain connected regardless of window state.

**Where is server state stored?**

Peer public keys and assigned IP addresses are persisted to ~/.rustguard/state.json. The server reloads this file on startup, so enrolled peers survive restarts without re-running rustguard join.

**Is the userspace daemon interoperable with the official WireGuard tools?**

The rustguard up wg.conf path implements the standard WireGuard Noise_IKpsk2 handshake and wire format and should be interoperable. The zero-config enrollment path (serve/join) uses a custom enrollment protocol on top and is not interoperable with other WireGuard implementations for the enrollment step itself; once enrolled, the resulting tunnel uses the standard protocol.
