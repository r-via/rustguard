## User

Section "ADR-007 — `subtle::ConstantTimeEq` for Cryptographic Comparisons" from `.anatoly/docs/02-Architecture/04-Design-Decisions.md`:

**Status:** Accepted (security improvement)

**Context:** Hand-rolled constant-time comparison functions using std::hint::black_box to suppress LLVM optimizations are fragile. Compiler behaviour is not specified and optimizations can change across Rust releases.

**Decision:** Replace all hand-rolled constant-time comparisons with subtle::ConstantTimeEq. The subtle crate is the de facto standard in the Rust cryptography ecosystem and provides compiler-portable guarantees.

**Consequences:**
- Timing side-channel risk is delegated to a well-reviewed dependency.
- Internal implementations are removed; maintenance burden is reduced.
