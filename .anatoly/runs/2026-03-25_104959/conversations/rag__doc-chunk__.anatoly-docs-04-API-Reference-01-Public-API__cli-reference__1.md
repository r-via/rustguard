### rustguard up

Starts a WireGuard tunnel from a standard wg.conf-formatted configuration file.

Reads peer definitions, assigns AllowedIPs routes (IPv4 and IPv6 CIDR), configures a TUN interface, and enters the tunnel loop. Handles SIGTERM/SIGINT for clean shutdown with route cleanup.

### rustguard serve

Starts a WireGuard server with the zero-config enrollment protocol. Enrollment begins **closed**; use [rustguard open](#rustguard-open) to admit clients.

Server state (peer keys, assigned IPs) is persisted to ~/.rustguard/state.json and survives restarts.

### rustguard join

Enrolls a client into a running rustguard serve instance. Generates a local keypair, performs the token-encrypted key exchange, and receives an assigned IP from the server's pool.

No configuration files are written or required. The tunnel starts immediately after successful enrollment.

### rustguard open

Opens the enrollment window on a running server for a specified number of seconds. Uses the UNIX domain control socket established by rustguard serve.

The window closes automatically via an atomic timestamp expiry. Existing peers are unaffected.

### rustguard close

Closes the enrollment window immediately on a running server.

No arguments. Communicates with the server via UNIX domain control socket.
