## User

Section "Transport Session" from `.anatoly/docs/02-Architecture/02-Core-Concepts.md`:

### Session

A pair of ChaCha20-Poly1305 keys (send and receive) derived at the end of a successful Noise_IKpsk2 handshake. Each session has an independent nonce counter for each direction.

### Replay Window

A 2048-bit sliding window bitmap that tracks received packet counters. Protects against replay attacks. The window check and window update are deliberately split into check() and update() operations: the window is only advanced **after** AEAD verification succeeds, preventing garbage counters from advancing state.

### Rekey

Sessions are rekeyed after 120 seconds or 2⁶⁰ messages, whichever comes first. The timer state machine initiates a new Noise handshake; existing transport continues until the new session is established.
