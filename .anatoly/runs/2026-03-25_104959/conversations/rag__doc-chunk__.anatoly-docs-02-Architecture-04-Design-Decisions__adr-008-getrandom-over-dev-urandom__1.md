## User

Section "ADR-008 — `getrandom` over `/dev/urandom`" from `.anatoly/docs/02-Architecture/04-Design-Decisions.md`:

**Status:** Accepted (security improvement)

**Context:** Direct reads from /dev/urandom via the file descriptor API bypass the operating system's entropy-readiness guarantees on some Linux configurations (early boot, containers) and are not portable to non-Linux targets.

**Decision:** Replace all direct /dev/urandom reads with the getrandom crate, which calls the getrandom(2) syscall on Linux and the appropriate OS-native API on other platforms.

**Consequences:**
- Entropy reads block until the kernel's random pool is initialised.
- CSPRNG seeding is correct on all supported platforms.
