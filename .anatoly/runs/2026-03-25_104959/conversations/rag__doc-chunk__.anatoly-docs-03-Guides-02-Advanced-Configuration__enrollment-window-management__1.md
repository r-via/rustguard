Section "Enrollment Window Management" from `.anatoly/docs/03-Guides/02-Advanced-Configuration.md`:

By default the enrollment window is **closed** on server start. Physical-presence authorization is enforced via a UNIX domain control socket. The window is opened for a bounded duration and then closes automatically via an atomic timestamp expiry.

**Recommendations:**

- Keep windows short (≤ 120 s) in untrusted environments.
- Run rustguard close immediately after the expected peer joins rather than waiting for the timer.
- rustguard status is non-destructive and safe to poll; it reports both the open/closed state and the number of enrolled peers.

The open/close/status commands communicate with the running daemon over the local control socket. They have no effect on already-established transport sessions — existing peers continue forwarding traffic regardless of window state.
