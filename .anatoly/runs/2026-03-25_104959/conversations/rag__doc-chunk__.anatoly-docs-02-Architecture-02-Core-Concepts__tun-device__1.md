Section "TUN Device" from `.anatoly/docs/02-Architecture/02-Core-Concepts.md`:

### TUN

A virtual network interface that exchanges raw IP packets with userspace. RustGuard reads plaintext IP packets from the TUN, encrypts them, and sends them over UDP; received UDP datagrams are decrypted and written back to TUN.

- **macOS**: utun device created via raw kernel control sockets.
- **Linux**: /dev/net/tun opened with IFF_TUN | IFF_NO_PI.

### Multi-Queue TUN

Linux TUN supports multiple file-descriptor queues attached to a single interface, enabling parallel processing across CPU cores without a single-queue bottleneck.

### AF\_XDP

Linux kernel bypass socket family. Allows RustGuard to exchange packets directly between NIC hardware queues and userspace memory (UMEM), bypassing the kernel network stack entirely.

### io\_uring

Linux asynchronous I/O interface. Used in rustguard-tun to issue TUN reads and UDP socket operations without blocking kernel transitions per packet.
