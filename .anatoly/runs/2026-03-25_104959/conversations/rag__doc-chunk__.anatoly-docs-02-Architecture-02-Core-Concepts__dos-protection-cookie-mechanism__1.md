Section "DoS Protection (Cookie Mechanism)" from `.anatoly/docs/02-Architecture/02-Core-Concepts.md`:

### MAC1

A 16-byte BLAKE2s MAC over the handshake message, keyed with the recipient's static public key. Always present. Verified **before** any DH operations to prevent CPU exhaustion from unauthenticated packets.

### MAC2

A 16-byte BLAKE2s MAC keyed with the current cookie value. Required only when the responder is under load. Proves the initiator can receive packets at the source IP address (return-routability check).

### CookieChecker

Server-side component that maintains a rotating secret and generates/validates cookies. The rotating secret changes on a timer to limit cookie validity windows.

### CookieState

Client-side component that stores the most recently received cookie and applies it as MAC2 in subsequent handshake initiations when prompted by the server.
