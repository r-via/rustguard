## User

Section "rustguard-crypto" from `.anatoly/docs/04-API-Reference/03-Types-and-Interfaces.md`:

**Crate:** rustguard-crypto · **no_std compatible**

### X25519KeyPair

Holds a Curve25519 private/public key pair used for the Noise_IKpsk2 handshake.

The private key bytes are zeroed on drop via ZeroizeOnDrop.

### X25519PublicKey

A peer's public key transmitted in handshake messages.

### ChaCha20Poly1305Key

A 256-bit symmetric key for AEAD encryption of transport packets.

Implements ZeroizeOnDrop. The encrypt() method on this type returns Option> rather than panicking on nonce exhaustion — the caller must rotate the session when None is returned.

### XChaCha20Poly1305Key

Extended-nonce (192-bit) variant used specifically for cookie encryption and the enrollment token protocol.

### TAI64N

An International Atomic Time (TAI64N) timestamp used in handshake initiations to prevent replay attacks. The format matches the WireGuard wire spec: 8-byte TAI64 seconds + 4-byte nanoseconds.

TAI64N values embedded in handshake initiations are validated against the most recently accepted timestamp for a peer; stale timestamps are rejected.

### Blake2sHash

Output of an HMAC-BLAKE2s or plain BLAKE2s operation. Uses RFC 2104 double-hash (ipad/opad) when produced by the HMAC path.

### HkdfOutput

Key material derived via HKDF over BLAKE2s.
