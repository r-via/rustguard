## User

Section "ADR-014 — UNIX Domain Socket Control Plane with Timed Enrollment Window" from `.anatoly/docs/02-Architecture/04-Design-Decisions.md`:

**Status:** Accepted

**Context:** Leaving enrollment permanently open means any party that knows the token can join at any time, including after the initial setup period. A Zigbee-style physical-presence model — open for a bounded time window, then closed — limits exposure.

**Decision:** The server exposes a UNIX domain socket. Enrollment is closed by default. rustguard open  opens the window for a bounded duration; rustguard close closes it immediately. The window state is tracked via an atomic timestamp. Existing peer sessions are unaffected by window state.

**Consequences:**
- Enrollment is only possible during a window explicitly opened by an operator with local access to the server.
- The attack surface for token-based enrollment is time-limited.
- The UNIX socket is only accessible to processes with filesystem permission on the socket path (i.e., local users).
