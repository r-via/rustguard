Section "rustguard-enroll" from `.anatoly/docs/04-API-Reference/03-Types-and-Interfaces.md`:

**Crate:** rustguard-enroll

### EnrollmentToken

A shared secret used to derive the XChaCha20-Poly1305 key that protects the enrollment key exchange. Supplied via --token on both server and client.

The token is never transmitted in plaintext; it is hashed with HKDF to produce the session encryption key.

### IpPool

CIDR-based address allocator. The server address is fixed at the network's .1; clients receive sequential addresses starting at .2.

### ServerState

Persisted server state written to ~/.rustguard/state.json on every peer enrollment and on clean shutdown.

### PeerRecord

A single enrolled peer entry within ServerState.

### EnrollmentWindow

Runtime state for the Zigbee-style physical-presence pairing window.

Window state is managed via an atomic timestamp to avoid locking in the hot path. Existing peers are unaffected when the window closes.
